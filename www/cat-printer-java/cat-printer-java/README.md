# cat-printer-java

A Java 17 port of the Cat-Printer Python driver. Provides a programmatic API for
JVM applications to drive the supported BLE thermal printers (`SC03h`, `MXTP-*`,
`GB0X`, `GT01`, `MX0X`, etc.) without the Python HTTP server in between.

This is a **library**, not an HTTP server. There is no UI, no IPP/CUPS, no
ImageMagick shell-out. Consumers call into `CatPrinter` directly.

## Status

- Pure-Java port of the printer command protocol, model registry, bitmap
  pipeline, PF2 font parser, and text canvas.
- BLE transport defined as an SPI (`BleTransport`).
- **Windows** is supported out of the box via a small bundled WinRT helper
  exe — see [windows-helper/README.md](windows-helper/README.md). On Windows,
  `BleTransport.platformDefault()` returns a working `WindowsTransport`.
- **Linux**: a starter backend against `com.sputnikdev:bluetooth-manager`
  lives at `examples/linux-bluez/BluezTransport.java`. Not on the build path —
  copy it into your own project, add the dependency, and verify against your
  hardware.
- **macOS**: no built-in backend. Implement `BleTransport` yourself (e.g.,
  via JNI to Core Bluetooth) and pass it to `new CatPrinter(transport)`.

## Build

```sh
cd cat-printer-java
./gradlew build
```

(You'll need to add a Gradle wrapper if one isn't present — `gradle wrapper`
from any local Gradle install will do it.)

### Windows helper

The Windows backend bundles `windows-helper/dist/cat-printer-ble-helper.exe`
into the published jar. The committed binary is what you ship; rebuild it
only if you change `windows-helper/Program.cs`. Rebuilding requires the
.NET 8 SDK on a Windows machine — see
[windows-helper/README.md](windows-helper/README.md). If the exe is missing,
the Gradle build still succeeds but logs a warning, and `WindowsTransport`
will fail at runtime.

## Usage

```java
import com.catprinter.CatPrinter;
import com.catprinter.ble.BleDevice;
import com.catprinter.ble.BleTransport;

import java.nio.file.Files;
import java.nio.file.Path;
import java.time.Duration;
import java.util.List;

public class Demo {
    public static void main(String[] args) throws Exception {
        // On Windows, platformDefault() returns a WindowsTransport that
        // drives the bundled WinRT helper. On other OSes, it throws and you
        // need to provide your own BleTransport (e.g. the Linux starter at
        // examples/linux-bluez/BluezTransport.java).
        try (BleTransport transport = BleTransport.platformDefault();
             CatPrinter printer = new CatPrinter(transport)) {

            List<BleDevice> devices = printer.scan(Duration.ofSeconds(4));
            if (devices.isEmpty()) {
                System.err.println("No printers found");
                return;
            }

            BleDevice sc03h = devices.get(0);
            printer.connect(sc03h);
            printer.setEnergy(0x4000);
            printer.setSpeed(36);

            byte[] pbm = Files.readAllBytes(Path.of("dump.pbm"));
            printer.printBitmap(pbm);
        }
    }
}
```

## API surface

`com.catprinter.CatPrinter`
- `scan(Duration)`, `scan(Duration, String modelFilter)` — discover printers.
- `connect(BleDevice)` — connect to a specific device.
- `printBitmap(byte[] | InputStream)` — print a PBM (P4) image. Width must
  match the model's `paperWidth` (384 for all current models).
- `printText(String, TextOptions, byte[] pf2FontData)` — render text via a
  PF2 font and print.
- `setEnergy(int)`, `setSpeed(int)`, `setFlip(boolean, boolean)`,
  `setMtu(int)` — tune output.

`com.catprinter.ble.BleTransport`
- SPI for BLE backends. Use `BleTransport.platformDefault()` on Windows;
  implement it yourself elsewhere. Linux starter:
  `examples/linux-bluez/BluezTransport.java`.

`com.catprinter.ble.windows.WindowsTransport`
- Windows backend. Spawns the bundled WinRT helper exe and routes
  scan/connect/write/notify over a JSON stdio protocol.

`com.catprinter.command.Commander`
- Lower-level access if you need to send raw commands. Uses the canonical TX
  characteristic `0000ae01-0000-1000-8000-00805f9b34fb`.

## Verification

Unit tests cover the deterministic pieces:

- `Crc8Test`, `BitOpsTest` — primitives match the Python implementation
  bit-for-bit.
- `ModelRegistryTest` — exact-match-then-longest-prefix lookup; `SC03h` and
  `MXTP-01` resolve correctly.
- `CommanderTest` — every command's byte sequence checked against goldens
  generated by the Python driver.
- `PrinterDataTest` — PBM read, vertical/horizontal flip, round-trip.
- `Pf2FontTest` — magic header validation.
- `HelperJsonTest` — round-trips the Windows helper protocol's JSON encoder
  and parser against fixtures.
- `WindowsTransportTest` — confirms `platformDefault()` still throws on
  non-Windows OSes.

Hardware verification: write a `Demo.java` like the snippet above and run with
the printer powered on. On Windows this works out of the box (with the helper
exe built); on Linux you must wire up `BluezTransport` from the example.
There's no automated end-to-end test in CI — BLE requires hardware.

## Caveats

- The sputnikdev `bluetooth-manager` artifact was last released around 2020.
  If it fails against modern BlueZ, swap in `com.github.hypfvieh:bluez-dbus`
  and adjust `BluezTransport` accordingly.
- Single connection at a time. The library is not thread-safe across multiple
  printers in one process.
- The `printText` API requires you to load a PF2 font yourself (e.g., the
  `unifont.pf2` shipped with the Python repo). The font format is GRUB's PF2.
- Windows helper adds ~30 MB to the jar (self-contained .NET 8 single-file).
  If size is a concern, switch the helper to native AOT — see
  `windows-helper/README.md`. The Windows machine itself needs no .NET install
  (the exe is fully self-contained); only the *build* machine needs .NET 8 to
  rebuild the helper.
